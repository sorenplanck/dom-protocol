## 🔴 Category A — the test that actually hung (runs by default, real RandomX)

| Test | File |
|------|------|
| `replay_two_independent_chains_converge` | `crates/dom-integration-tests/tests/replay_determinism.rs:258` |

This test performs **real RandomX mining of two independent chains** to prove replay determinism. It does **not** honor any fast-mining shortcut, so it cannot be sped up. In the audit sandbox it ran **>56 minutes without completing**; the maintainers' own VPS previously timed it out at 900 s. Its single-chain sibling `replay_same_chain_reopens_to_identical_tip` (line 318) **passed**.

Run it in isolation (expect a long runtime; use a fast dedicated host, ≥1 block/60 s):

```bash
cargo test -p dom-integration-tests --test replay_determinism \
  replay_two_independent_chains_converge -- --exact --nocapture
```

**Action required:** run this on dedicated hardware before relying on it as a release gate.

Run a single ignored test in isolation (example: `two_node`), with the env vars the release-blocker gate uses:

```bash
DOM_REGTEST_FAST_MINING=1 DOM_NETWORK=regtest \
  cargo test -p dom-integration-tests --test two_node -- --ignored --nocapture
```

Swap `two_node` for each file above. To run **all** ignored tests across the workspace at once:

```bash
DOM_REGTEST_FAST_MINING=1 DOM_NETWORK=regtest \
  cargo test --workspace -- --ignored --test-threads=1
```

## Important distinction

- **Category A** (`replay_two_independent_chains_converge`) does real RandomX mining and **ignores** the fast-mining flag — it will be slow regardless of environment.
- **Category B** tests were blocked because of slow RandomX prologue under WSL; they **likely honor** `DOM_REGTEST_FAST_MINING=1`, so they may run quickly on a normal host with that flag set. The `env-blocked-wsl` label is an environment limitation, **not** a protocol bug — any non-environment failure on a dedicated host is a real bug to investigate.
