SECURITY BOUNDARY — DOM REPOSITORY

You are running with full filesystem execution permissions, but you are authorized to operate only inside the current DOM repository.

Repository boundary:
- Do not read files outside this repository.
- Do not write files outside this repository.

git config --get core.hooksPath

You must not modify protected files/directories unless the user explicitly authorizes it in the prompt.

If a protected file appears necessary, stop and report:
1. exact file;
2. why it seems necessary;
3. expected protocol risk;
4. minimal proposed change;
5. validation required.
