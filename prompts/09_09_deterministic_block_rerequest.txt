TASK 09/50 — IMPLEMENT DETERMINISTIC BLOCK RE-REQUEST

Problem:
If blocks are dropped due to missing parents or ordering gaps, the node needs deterministic recovery instead of hoping for peer retransmission.

Objective:
Implement a deterministic re-request mechanism for missing blocks.

Tasks:
1. Detect missing parent/header/body situations explicitly.
2. Schedule re-request by hash and/or height.
3. Use deterministic peer selection where possible, or stable ordering of candidate peers.
4. Apply bounded retry and backoff without consensus side effects.
5. Avoid duplicate request storms.
6. Add logs for missing block, requested peer, retry count, and outcome.
7. Add tests:
   - missing parent triggers request;
   - duplicate missing parent does not trigger unbounded requests;
   - parent arrival drains dependent orphans;
   - delivery order does not change final tip/state.
